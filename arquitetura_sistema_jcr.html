Como o sistema funciona

Fluxo Completo do Sistema

Do upload do arquivo SPED até o relatório final por cliente — cada etapa do processamento e onde cada tecnologia atua.

1
Cadastro do cliente
Analista cadastra o cliente no sistema com CNPJ, regime tributário, atividade e período a analisar. Dados salvos no Supabase.
HTML/JS Supabase Tabela: clientes
2
Upload dos arquivos SPED
Analista faz upload do SPED Fiscal e/ou SPED Contribuições diretamente pela interface. Arquivos são enviados para o backend Python.
HTML/JS — drag & drop Python — recebe o arquivo EFD Contribuições / SPED Fiscal
3
Processamento Python — parsing do SPED
Python lê o arquivo pipe-delimitado, extrai os registros por tipo (C100, C170, A100, A170, E110 etc.) e monta as estruturas de dados para análise.
Python — parser SPED pandas Registros: C100, C170, A100, A170, E110
4
Análise automática por tese
Para cada tese habilitada, Python aplica as regras de identificação: cruza CST, CFOP, NCM e valores. Calcula o potencial de recuperação por período e por tese.
Python — motor de regras ADM: 6 teses JUD: 5 teses pandas + numpy
5
Salvamento no banco de dados
Resultado da análise salvo no Supabase por cliente, por tese e por período. Permite histórico e comparação entre análises.
Supabase — PostgreSQL Tabela: oportunidades Tabela: analises
6
Dashboard de oportunidades
Interface exibe as oportunidades encontradas por cliente — valor estimado por tese, status ADM ou JUD, e próximos passos recomendados.
HTML/JS — dashboard Supabase — leitura Chart.js — gráficos
7
Geração de relatório para o cliente
Sistema gera relatório em PDF ou HTML com as oportunidades identificadas, valores estimados e orientação sobre ADM vs. JUD — pronto para apresentar ao cliente.
Python — geração PDF HTML — relatório visual reportlab / weasyprint

Tecnologias escolhidas

Stack Tecnológico

Cada tecnologia escolhida por um motivo específico — familiar, gratuita onde possível, e adequada ao problema.

🐍
Python — Backend
Processamento dos arquivos SPED, parser pipe-delimitado, aplicação das regras de cada tese, cálculo dos créditos e geração de relatórios PDF. Você já tem experiência com Python.
pandas numpy fastapi reportlab supabase-py uvicorn
🌐
HTML + JavaScript — Frontend
Interface que os analistas usam no navegador. Upload de arquivos, visualização de oportunidades, dashboard e geração de relatórios. Mesmo padrão visual já construído.
HTML5 CSS3 JavaScript Chart.js Fetch API
🗄️
Supabase — Banco de Dados
PostgreSQL gerenciado com autenticação de usuários, armazenamento de clientes, análises e oportunidades. Você já usa no App do Rafael — caminho conhecido.
PostgreSQL Supabase Auth Row Level Security Realtime Storage
🚀
Infraestrutura
Frontend no GitHub Pages (gratuito). Backend Python no Railway ou Render (gratuito até certo uso). Supabase free tier para começar — migra para pago quando crescer.
GitHub Pages Railway Supabase Free GitHub Actions
Ordem de desenvolvimento

Roadmap de Desenvolvimento

Construir por partes — cada etapa já entrega valor antes da próxima começar.

1
Fundação — Parser SPED e banco de dados
Criar o parser Python que lê arquivos SPED pipe-delimitados e extrai os registros. Configurar o Supabase com as tabelas de clientes e análises. Testar com arquivos reais dos seus clientes.
Parser SPED Python Extração C100/C170 Extração A100/A170 Supabase — tabelas Testes com SPED real
⏱ Estimativa: 2 a 3 semanas
2
Primeira tese — EFD x SPED (PIS/COFINS)
Implementar o cruzamento entre SPED Fiscal e SPED Contribuições — primeira e mais usada tese ADM. Validar com casos reais antes de avançar para as demais.
Cruzamento EFD x SPED Filtro por CST Cálculo do crédito Salvar oportunidades Validação com caso real
⏱ Estimativa: 1 a 2 semanas
3
Interface básica — upload e resultados
Tela de upload do SPED, lista de clientes e visualização das oportunidades identificadas. Autenticação com Supabase Auth para controlar o acesso dos analistas.
Tela de login Upload de arquivo Lista de oportunidades Autenticação Gestão de clientes
⏱ Estimativa: 2 semanas
4
Demais teses ADM
Implementar as outras 5 teses administrativas: Exclusão ICMS PIS/COFINS, ICMS-ST, GLP/Diesel, Monofásicos SN e PGDAS SN. Cada tese é um módulo independente.
Exclusão ICMS PIS/COFINS ICMS-ST GLP/Diesel LC 192 Monofásicos SN PGDAS SN
⏱ Estimativa: 3 a 4 semanas
5
Dashboard e relatório PDF
Dashboard com gráficos de oportunidades por cliente e por tese. Geração do relatório em PDF para apresentar ao cliente com o diagnóstico completo.
Dashboard Chart.js Totalizadores Geração PDF Python Relatório por cliente Export HTML
⏱ Estimativa: 2 semanas
6
Teses JUD + gestão de ações
Implementar as 5 teses judiciais como análise de potencial (sem transmissão — apenas diagnóstico). Módulo de acompanhamento de ações em andamento com advogado parceiro.
5 teses JUD Gestão de ações Status por processo Controle de honorários Alertas de prazo
⏱ Estimativa: 3 semanas
